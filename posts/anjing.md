# Quietly

`Quietly` 是一个面向 `Minecraft 1.21.1 ~ 1.21.10` 的服务端静默开箱模组，支持 `Fabric` 和 `NeoForge`。

它的目标很简单：

- 让玩家通过 `Shift + 双击右键` 启动静默开箱
- 用 `Bossbar` 显示开启进度
- 保留容器动画
- 不播放原版容器开关音效
- 不触发原版 `CONTAINER_OPEN / CONTAINER_CLOSE` 声感事件

## 功能特性

- 服务端主导，服务器必须安装，客户端可选
- 支持通用带界面容器的静默开启流程
- 开启耗时按容器内“物品种类数”计算
- 保留箱子、木桶、末影箱、潜影盒等容器动画
- 屏蔽原版容器开关声音
- 屏蔽原版容器开关相关声感事件
- 内置中英文本地化

## 使用方式

1. 对准一个带界面的容器。
2. 按住 `Shift`。
3. 切到主手空手状态，对同一容器快速双击右键以启动静默开启。
4. 持续保持潜行直到 `Bossbar` 读满。
5. 读满后容器会以静默方式打开。

## 当前行为

- 触发方式仅限 `Shift + 主手双击右键`
- 仅在 `主手空手` 时触发，避免影响潜行放置方块/物品
- 开启过程中必须持续潜行
- 松开潜行、离开交互范围、提前打开其他界面都会中断
- 开启耗时公式为 `12 + 物品种类数 * 4` tick
- 物品种类数按容器内非空 `Item` 去重计算，不按堆叠数量计算

## 支持范围

- 所有能提供菜单的通用容器
- 已专门处理动画与静音链路的原版容器：
- 普通箱子 / 陷阱箱
- 木桶
- 末影箱
- 潜影盒
- 其他带界面方块如熔炉、工作台、漏斗等同样可走静默开启逻辑

## 兼容范围

- Minecraft: `1.21.1 ~ 1.21.10`
- Fabric Loader: `0.19.2`
- Fabric API: `0.116.12+1.21.1`
- Java: `21`
- `1.21.11` 不在当前支持范围内

## 项目结构

- `common/`: 公共逻辑、静音状态机、通用 Mixin
- `fabric/`: Fabric 入口与平台资源
- `neoforge/`: NeoForge 入口与平台资源

## 构建

构建 Fabric：

```bash
./gradlew.bat :fabric:build
```

构建 NeoForge：

```bash
./gradlew.bat :neoforge:build
```

构建完成后可在对应平台的 `build/libs/` 目录查看产物。

- 当前实现已经收口到 `1.21.1 ~ 1.21.10`
